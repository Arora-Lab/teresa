# Contributing Guidelines

We welcome contributions from volunteers, developers, and designers.

## Getting Started
1. Review `PROJECT.md` and `Architecture.md` to understand the project structure.
2. Read the `CodingStandards.md` and `UIStyleGuide.md`.
3. Check the GitHub Projects board for issues marked "Ready".

## Making a Contribution
1. Fork the repository.
2. Create a feature branch following the `Branching.md` guidelines.
3. Write your code, ensuring tests pass and no linter errors remain.
4. Submit a Pull Request using the provided PR template.

## Review Process
- PRs require at least one approving review.
- UI changes must include screenshots or recordings in the PR description.
- Ensure localization (`vi.json` and `en.json`) is complete for any new features.
